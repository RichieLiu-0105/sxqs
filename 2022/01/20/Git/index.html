Git

Git：分布式版本控制系统。C语言编写而成。

1.安装之后

2.创建版本库

版本库又名仓库，英文名：repository，可简单理解成一个目录，这个目录里的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以跟踪历史，或者在将来某个时刻可以“还原”。

备注：所有的版本控制系统，其实只能跟踪文本文件的改动（修改）。

3.时光机穿梭

工作区修改—>暂存区git add—>分支git commit

4.远程仓库

GitHub提供Git仓库托管服务。

  • 在GitHub上，可以任意Fork开源仓库；
  • 自己拥有Fork后的仓库的读写权限；
  • 可以推送pull request给官方仓库来贡献代码。

本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，需要设置：

  1. 创建SSH Key：ssh-keygen -t rsa -C "youremail@example.com"
  2. GitHub 添加SSH Key：id_rsa.pub公钥文件内容#确定是你推送的
  3. 本地与远程仓库关联：git remote add origin git@github.com:Github账户名/仓库名.git #远程库默认名字origin
  4. 本地推送到远程：git push origin master #第一次-u参数，做分支关联
  5. 删除远程库：git remote rm <name> #解除本地和远程的绑定关系，并非物理上删除了远程库，要真正删除远程库，需要登录Github删除。
  6. 查看远程库信息：git remote -v
  7. 远程库克隆到本地：git clone git@github.com:Github账户名/仓库名.git

5.分支管理

平行宇宙互不打扰。主分支默认称为master分支（master指向提交，HEAD指向master，也就是当前分支）

  1. 创建并切换分支：git checkout -b 分支名/git switch -c 分支名，相当于两个命令：git branch 分支名#创建分支，git checkout 分支名/git switch#切换分支
  2. 查看所有分支：git branch，当前分支前面会标一个*号
  3. 合并指定分支到当前分支：git merge 分支名
  4. 删除分支：git branch -d 分支名
  5. 当分支冲突时，先解决冲突后，再提交，git log --graph可以看到分支合并图
  6. 合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。
  7. git stash#临时存储未提交的分支， git stash list#查看临时存储未提交的分支，git stash apply#恢复但stash不删除，需要再用git stash drop来删除，或者直接用git stash pop，恢复的同时把stash内容也删了。git cherry-pick <commit> 可以复制一个特定的提交到当前分支，避免重复劳动。
  8. 删除未合并过的分支：通过git branch -D <name>强行删除。
  9. 远程库查看：git remote或者git remote -v，从本地库推送到远程库分支：git push origin master，如果推送失败，先用git pull抓取远程的新提交。本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；建立本地分支和远程分支的关联，使用git branch --set-upstream branch-name origin/branch-name，从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。
  10. git rebase可以把本地未push的分叉提交历史整理成直线，目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方对比。

6.标签管理

tag就是一个让人容易记住的有意义的名字，它跟某个commit绑在一起

  1. 创建标签：`git tag ``
  2. ``git show `#查看标签信息
  3. git tag -a <tagname> -m "blablabla..."可以指定标签信息
  4. 命令git push origin <tagname>可以推送一个本地标签
  5. 命令git push origin --tags可以推送全部未推送过的本地标签
  6. 命令git tag -d <tagname>可以删除一个本地标签
  7. 命令git push origin :refs/tags/<tagname>可以删除一个远程标签

7.自定义Git

  1. git config --global color.ui true #git显示颜色
  2. 忽略特殊文件：编写.gitignore
