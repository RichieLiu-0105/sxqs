《微信互联网平民创业》

主题：如何赚钱

你把微信当作什么

微信2011年1月21日发布，借助腾讯的用户基础，允许QQ和电话号码注册（同时允许上传通讯录），获得大量用户，几乎干掉了运营商的短信业务。

2020年微信月活用户已经超过12亿人，安装率100%。不再仅仅是把它当作一个替代运营商所提供的短信功能的免费替代品而已了，现在的微信干脆是个完整的互联网

你应该把微信当作什么

微信是一个经济世界，微商，只不过是「在微信这个经济世界里从事商业的人群」而已。微信是我们的赚钱工具。

微信提供了什么

Google：通过关键词搜索技术，它把互联网上的每一篇文章中的几乎每个关键词都变成了它的广告牌……

微信：给每一个用户 提供了一个竟然是免费的却又可以长期打造、长期积累的「屏幕」（或者准确地讲，「广告屏幕」）……

很多的时候，真的不是我们不努力，不是我们不勤奋，也不是我们不聪明，甚至不是我们不动脑，更不是我们没有脑子…… 只不过是我们「没看到」而已。一旦看到了，脑子就会懂；脑子开始动并且不停地动，人就会变聪明；人变聪明了，就会开始琢磨；开始琢磨了，就不可能不行动；开始行动了，就不可能不改变…… 就这么简单 —— 所有的一切，「开眼」都是起点。所以古人总是说，「读万卷书不如行万里路」—— 行万里路为什么竟然更有效啊？因为行万里路的过程中必然反复经历「开眼」，即，所谓的「长见识」。

我们自己最缺的是什么

机会，真正有本事的人不愁没机会，任何时候都不要害怕错过机会，如果你竟然错过了，那只能说明事实上你并在没有驾驭那个机会的能力而已。

本钱，无本生意，要想尽一切办法从「一份时间只能出售一次」升级到「同一份时间可以重复出售很多次」。

假设你想经商 —— 在一个商业社会里生存，经商好像是最佳选择之一，那就做个商人呗 —— 那么，你缺的根本不是机会，缺的也根本就不一定是本钱…… 可你最终并没有成为一个商人，更没有成为一个好商人，到底是因为缺了什么？两个字：能力；四个字：商业能力。

商业能力：识别长期价值的能力。无论做什么事情，都要事先认真问自己，并且还要真诚回答自己，哪怕就这么一句：这事儿值得做很久吗？。既然它能做很久，也值得做很久，那么，「我应该付出的是什么？」公平交易。同时趋势这个东西，从来都不是用来驾驭的，从来都应该是顺应的，选择比努力更重要，决策能力是商业能力最重要的。

流量是流量，变现是变现，它们俩常常并不是一回事儿……

《把时间当作朋友》

关键词：心智、思考力、积累、时间、成长、理性、现实、坚强

短语

  1. 有些人在有些事上毫无功利心，这有什么奇怪的吗？

答案

  1. 要管理的不是时间，而是自己
  2. 心智 ->思考->观察->记录->总结->分享->行动，用正确的方式做正确的事情，什么是是正确的方式，什么事情是值得去做，判断标准是是否现实
  3. 只有学会正确地思考，才意味着真正进化成人
  4. 有些认识，哪怕是简单的常识，也需要亲身经历后才能真正体会（心智真正成熟的人在一些情况下能够做到无须亲自经历，仅凭思考就得到深刻的体会。）
  5. 越是能力差的人，越有那种非常不现实又极其脆弱的完美主义倾向。他们不现实，是因为他们不懂。
  6. 每一时刻的现状都是过去某一或者某些时刻的结果，而每一时刻的现状都是未来某一或者某些时刻的原因。没有人能够逃脱现实的束缚。
  7. 学习是投资回报率最高的行为。
  8. 与其不停地寻找“更好的方法”，还不如马上开始行动。

章节

  1. 困境（时间不可管理，能管理的只有自己，自己的管理靠心智与积累）

    1. 问题：时间紧迫，时间不够用，自身及外界因素影响产生
    2. 慌乱：即勤奋又懒惰，被时间压力导致，错误的思考和选择
    3. 解决：时间不可管理，真正能够管理的是自己，而管理自己，必须开启心智，看清楚，想明白，问题出在自己身上。我们面临的问题与时间无太多关系，而是自己，自己解决方案就是一切靠积累。

  2. 醒悟（提升心智，利用知识和经验以及思考方法和模式的积累控制自己的大脑）

    1. 孰主孰仆：你要控制你的大脑，而不是被自己大脑左右，运用心智获得解救，不要让自己再跟着感觉走，成为大脑的奴隶（反思、思考）
    2. 何为心智：知识和经验的总和，也包括思考方法和思考模式，一点点积累，人与人的差距更多是心智而非智商

  3. 现实（接受-牢记-坚信-记录-反思-反省，改掉不现实的念头与影响）

    1. 速成绝无可能：告诉自己：我有不足，我需要时间，我没办法一蹴而就。（一口吃成大胖子）
    2. 交换才是硬道理：（我要的和我有的）交换，浮躁的根本：一在于积累少（我有的太少，无法交换），二在于无方法和经验（努力勤奋或投机，用什么方法或经验积累我有的）；（想要的太多，做的太少，要想完成想要的，就需要想方法去做完成我有的，才能获得想要的）
    3. 完美永远不存在：因为不懂，所以不现实；因为不现实，所以脆弱；因为脆弱，所以受伤；受伤因为要求太高，所以也总做不到。根因在于没有什么事情可以一下子就能做好，而是有能力做到更接近完美的人，并且，他们一直在努力。忍耐不完美，否则事情根本无法完成。（不要再追求完美才开始，完美主义是不断追求完美，而不是一下子完美）
    4. 未知永远存在：害怕未知，害怕不确定的东西，形态就是钻牛角尖，必须忍受一定的未知才能进步。在未知中不断前行，产生焦虑，进而步步错，先记下来问题，然后继续前行（不要再钻牛角尖，要学会接受短暂的未知）
    5. 现状无法马上摆脱：有什么做什么，有什么用什么，做什么都做好，用什么都用好，接受现状；（不要在“反正我的梦想不是这个”作为借口，努力积累当下的能力）
    6. 与时间做朋友：用正确的方法做正确的事，判断的标准是看它是否现实。一切愚蠢的行为都来自否定现实、逃避现实。学会坦然接受现实，即使痛苦（逃避与幻想，清心寡欲与自我幻想）

  4. 管理（心智提升方法：时间-行动-心态-抓重点-方式-记录-计划-流程-预演-验收）

    1. 估算时间：做任何事情之前先判断其熟悉程度（或陌生程度），再据此判断估算完成任务所需要的时间（切勿盲目估算）
    2. 及时行动：不行动本质是恐惧，在意别人看法，害怕失败，只有接受现实，才能勤能补拙（现在行动就是最好的时候）
    3. 直面困难：直面解决困难问题，不回避苦难问题，切勿专注简单，回避困难（创意的事情1%和枯燥的事情99%）
    4. 关注步骤：what、why、how三思法则，how最重要！做任何事情，学会思考方式最重要！自己的项目经理（自主、独立是心智成熟的人必有得素质），通过关注方法反复拆分任务，变的现实、踏实。（学会思考，方法-拆分-执行）
    5. 并行串行：这个道理可以用在什么地方，举一反三（提高效率）
    6. 感知时间：事件-时间日志（记录过程记录）
    7. 记录开销：记录天、周、月、季做得事情（记录时间开销）
    8. 制定预算：任务重要程度判断标准是这项任务的完成是否确实对目标达成有益（每天制作时间预算）
    9. 计划：目标可现实可行，长期计划是需要通过实践才能习得，有些时候没必要做计划（目标要可行，最重要是马上行动）
    10. 列表：最重要的任务评判标准是那个真正对目标的实现有帮助的任务。（任务可实现，坚持到底）
    11. 流程：为常见任务制定流程，梳理流程，思考细节（重复事情，马上总结，整理，搞清楚流程，提高效率，减少失误）
    12. 预演：万事皆可提前准备，万事皆需提前准备（提前准备，预演流程）
    13. 验收：为每一次任务设置相应的验收，注意更多细节，进行更多思考、并不由自主的专注（验收结果）

  5. 学习（学习是最高的回报，保持好心态，切勿经验局限，学会检索、阅读、思考、写作、实践）

    1. 效率本质：学习是投资回报率最高的行为。（不断学习才是对自己最大的投资）
    2. 基本途径：获取更多知识几乎是我们开拓自身心智的唯一手段。（体验-试错）-观察-阅读（阅读才是最重要得）
    3. 主要手段：思考，正确的思考，才是获取真正意义上的知识的主要手段。（正确思考，更值得我们投入时间与精力。）
    4. 经验局限：切勿经验主义，科学的思考（切勿经验主义）
    5. 自学能力：自学基础是阅读理解能力，检索能力是最基础能力，写作是自学占重要位置，实践是自学能力最终的转化，开放心态是提升心智的基础（检索-阅读-写作-实践，学习这个过程要保持开放心态）
    6. 学习进程：在学习的任何一个阶段，都会有一段时间进展缓慢。只有经历积累的过程，“量变到质变”的效果才会出现，才有可能突飞猛进。（学习并非线性，而是波浪起伏上升）

Git

Git：分布式版本控制系统。C语言编写而成。

1.安装之后

2.创建版本库

版本库又名仓库，英文名：repository，可简单理解成一个目录，这个目录里的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以跟踪历史，或者在将来某个时刻可以“还原”。

备注：所有的版本控制系统，其实只能跟踪文本文件的改动（修改）。

3.时光机穿梭

工作区修改—>暂存区git add—>分支git commit

4.远程仓库

GitHub提供Git仓库托管服务。

  • 在GitHub上，可以任意Fork开源仓库；
  • 自己拥有Fork后的仓库的读写权限；
  • 可以推送pull request给官方仓库来贡献代码。

本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，需要设置：

  1. 创建SSH Key：ssh-keygen -t rsa -C "youremail@example.com"
  2. GitHub 添加SSH Key：id_rsa.pub公钥文件内容#确定是你推送的
  3. 本地与远程仓库关联：git remote add origin git@github.com:Github账户名/仓库名.git #远程库默认名字origin
  4. 本地推送到远程：git push origin master #第一次-u参数，做分支关联
  5. 删除远程库：git remote rm <name> #解除本地和远程的绑定关系，并非物理上删除了远程库，要真正删除远程库，需要登录Github删除。
  6. 查看远程库信息：git remote -v
  7. 远程库克隆到本地：git clone git@github.com:Github账户名/仓库名.git

5.分支管理

平行宇宙互不打扰。主分支默认称为master分支（master指向提交，HEAD指向master，也就是当前分支）

  1. 创建并切换分支：git checkout -b 分支名/git switch -c 分支名，相当于两个命令：git branch 分支名#创建分支，git checkout 分支名/git switch#切换分支
  2. 查看所有分支：git branch，当前分支前面会标一个*号
  3. 合并指定分支到当前分支：git merge 分支名
  4. 删除分支：git branch -d 分支名
  5. 当分支冲突时，先解决冲突后，再提交，git log --graph可以看到分支合并图
  6. 合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。
  7. git stash#临时存储未提交的分支， git stash list#查看临时存储未提交的分支，git stash apply#恢复但stash不删除，需要再用git stash drop来删除，或者直接用git stash pop，恢复的同时把stash内容也删了。git cherry-pick <commit> 可以复制一个特定的提交到当前分支，避免重复劳动。
  8. 删除未合并过的分支：通过git branch -D <name>强行删除。
  9. 远程库查看：git remote或者git remote -v，从本地库推送到远程库分支：git push origin master，如果推送失败，先用git pull抓取远程的新提交。本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；建立本地分支和远程分支的关联，使用git branch --set-upstream branch-name origin/branch-name，从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。
  10. git rebase可以把本地未push的分叉提交历史整理成直线，目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方对比。

6.标签管理

tag就是一个让人容易记住的有意义的名字，它跟某个commit绑在一起

  1. 创建标签：`git tag ``
  2. ``git show `#查看标签信息
  3. git tag -a <tagname> -m "blablabla..."可以指定标签信息
  4. 命令git push origin <tagname>可以推送一个本地标签
  5. 命令git push origin --tags可以推送全部未推送过的本地标签
  6. 命令git tag -d <tagname>可以删除一个本地标签
  7. 命令git push origin :refs/tags/<tagname>可以删除一个远程标签

7.自定义Git

  1. git config --global color.ui true #git显示颜色
  2. 忽略特殊文件：编写.gitignore

备注：以上整理学习来源于廖雪峰的官方网站

《进化式运营》

思考能力是一项核心竞争力。

产品负责生孩子，运营负责养孩子。

核心：用户

以用户视角解决问题：接触、认知、关注、体验、使用、付费、习惯、分享等

  1. 接触：用户调研（认知水平、渠道用户分布）、制作接触内容（吸引注意力、灌输价值、推荐关注渠道）
  2. 认知：接触用户（降低关注门槛、认知教育、锁定有点兴趣的用户群体）
  3. 关注：区分粉丝及订阅用户、新手用户、潜伏用户、产品用户
  4. 体验：新用户及体验用户、体验用户关注事项
  5. 使用：场景体验测试、促使付费决策
  6. 习惯：持续激励用户习惯（分析用户有效的刺激心理）
  7. 分享：提供分析噱头（多元化分享内容、把握分享契机）

各环节运营内容、手段：

  1. 接触-认知：策略（线上线下推广、新媒体）；手段（SEO、SEM、病毒营销）
  2. 认知-关注：策略（新媒体、内容、社群）；手段（微博、公号、社交、H5）
  3. 关注-体验：策略（内容、活动、用户）；手段（微博、公号、社群、内容、直播）
  4. 体验-使用：策略（产品、活动、用户）；手段（行为分析、用户调研、需求输出）
  5. 使用-付费：策略（活动、用户）；手段（数据分析、积分、电话）
  6. 付费-习惯：策略（用户、内容、产品）；手段（文案、广告、游戏化激励）
  7. 分享：策略（活动、用户、产品）；手段（社交货币、游戏化激励）

获取有效流量的具体运营步骤：

  1. 用户调查
  2. 内容制作
  3. 投放渠道
  4. 数据反馈
  5. 内容调优

运营人基础知识

一、学科知识：
  1. 心理学：社会心理学
  2. 传播学：传播规律、媒介
  3. 统计学：数据分析
  4. 营销学：营销理念和思维
  5. 经济学：商业社会
  6. 管理学：企业运作
  7. 进化学：认知
  8. 职场学：沟通艺术
二、思维模式：

  1. 用户思维

    一切以用户的需求为导向。抓住用户，你就拥有无限机会。

  2. 迭代思维

    创新速度与效率时代，不断根据用户需求对产品或服务进行调整升级。快速。

  3. 流量思维

    对业务运营的理解。流量就是金钱。有效流量。

  4. 社会化思维

    对传播链、关系链的理解，读懂社会化思维才能更好地适应动荡地市场局面。用网紧紧抓住你的用户。

互联网流量经济

最近越来越体会到互联网的魅力，以前总是看到的是一个点，根本不知其所以然，如今进入市场流量领域，真实的体会到了互联网的魅力以及互联网流量生意所带来的财富，这或许就是科技的力量😂

市场&流量&销售

当以生意的角度，去思考市场行为，你会发现原来看到的都只是一个点，对市场行情一无所知，单纯只是一颗螺丝钉。而互联网市场结果就是流量，流量等于用户，有了用户客流，通过销售行为，生意才能带来更多的利润。终究市场=利润，如何在市场竞争中活下来，需要的就是手段和思想。互联网改变了传统生意，让生意更加简单，只要你有想法，一切皆有可能。

技术&产品&服务

有了市场想法，然后依靠技术做出产品，通过产品及服务来变现。当然产品可以是任何形态，只要能解决用户的某一个需求，都会有一定市场价值利润的，只是利润的多与少问题。而技术的思维只考虑技术&产品的牛逼，就会带来市场价值利润，这是理想状态，而且及其不现实。因为同类产品很多，恶意竞争也很多，所以好的产品只是基础，市场玩法才能最大化价值利润。

总结

市场+技术：以增长黑客为目标，通过技术手段来实现市场占有（技术+想法）

流量+产品：以流量为结果，以产品为基础，全局思考（流量优化+产品反馈）

销售+服务：最后的转化！（ROI）
